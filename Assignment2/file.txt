Acceptance criteria:

1. It is possible to create users via a POST request.

2. It is possible to read (or view) users via a GET request.

3. It is possible to update users via a PUT request.

4. It is possible to delete users via a DELETE request.

5. All data entry operations must be validated to ensure an end user cannot purposely or accidentally submit invalid data.

6. As a bonus: distinguish between reading a single user or scanning (reading) all users!
